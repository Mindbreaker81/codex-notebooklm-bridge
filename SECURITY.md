# Security Policy

## Supported Versions

Security fixes apply to the current main branch.

## Reporting a Vulnerability

If you find a security issue, do not open a public issue with exploit details.

Preferred reporting path:

1. Open a private security advisory on the repository if GitHub support is available.
2. If that is not available, open a minimal public issue that only states the area affected and request a private contact method.

Please include:

- A clear description of the issue.
- The affected file or command path.
- Reproduction steps, if safe to share.
- Any evidence that helps confirm impact.

Do not include secrets, live tokens, or sensitive user data in reports.
